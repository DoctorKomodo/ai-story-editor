Stop maintaining a parallel HTML universe — make the codebase be the design system.

The live app is half-migrated: 18 components on tokens, 7 still on raw Tailwind, 4 dead orphans, 18 unverified. This package makes the 4 deletions trivial, the next 7 ports mechanical, the 18 unknowns auditable, and every future feature a copy-paste from @/design/primitives instead of a port from HTML.

design/primitives.tsx Modal · Button · Field · Input · Textarea · Pill · IconButton · Spinner. Drop-in replacements for the chrome StoryPicker hand-rolls inline. This is the only file Phase 1 actually adds to the repo.

04

Porting a component

Worked example: CharacterSheet.tsx — exercises every primitive (Modal, Field, Input, Textarea, Button × 3 variants, nested confirm dialog).
● Before — hand-rolled chrome + raw Tailwind
<div className="fixed inset-0 bg-black/40
    flex items-center justify-center z-50">
  <div role="dialog" aria-modal="true"
       className="bg-white rounded-md shadow-lg
         w-full max-w-2xl max-h-[90vh]">
    <form onSubmit={handleSubmit}>
      <h2>Edit character</h2>

      <label>
        <span>Name *</span>
        <input className="border
          border-neutral-300 rounded
          px-3 py-2 focus:ring-2
          focus:ring-blue-500" />
      </label>
      {/* …6 more identical labels… */}

      <button className="bg-red-600
          text-white hover:bg-red-700">
        Delete
      </button>
      <button className="bg-neutral-100
          text-neutral-800">Cancel</button>
      <button className="bg-blue-600
          text-white hover:bg-blue-700">Save</button>
    </form>
  </div>
  {/* + nested confirm dialog repeats
      the entire backdrop + card chrome */}
</div>
● After — primitives do the chrome
<Modal open={open} onClose={onClose}
       labelledBy={headingId} size="lg">
  <form onSubmit={handleSubmit}>
    <ModalHeader id={headingId}
                 title="Edit character"
                 onClose={onClose} />

    <ModalBody>
      <Field id={nameId} label="Name" required>
        <Input id={nameId} value={fields.name}
               onChange={onName} />
      </Field>
      {/* role / age / appearance /
          voice / arc / personality */}
    </ModalBody>

    <ModalFooter>
      <Button variant="danger"
              onClick={openConfirm}>Delete</Button>
      <div className="ml-auto flex gap-2">
        <Button variant="secondary"
                onClick={onClose}>Cancel</Button>
        <Button variant="primary"
                type="submit">Save</Button>
      </div>
    </ModalFooter>
  </form>

  <Modal open={confirmOpen} size="sm"
         role="alertdialog"
         dismissOnBackdrop={false}>
    {/* confirm body — same primitives */}
  </Modal>
</Modal>

What disappears: 4 × bg-white rounded-md shadow-lg backdrop blocks, 7 × hand-rolled label + input recipes, 6 × button color recipes (red / blue / neutral hover pairs), the z-50 / z-[60] stacking dance, and 80+ lines of focus-trap / Escape / click-outside boilerplate. Roughly 150 lines down to 70; once CharacterSheet ships, StoryModal and AccountPrivacyModal follow the exact same shape.

Substitution cheatsheet ~90% of swaps you'll hit

Camp B (Tailwind raw)Camp A (token)
bg-white→bg-bg-elevated
bg-neutral-50→bg-bg-sunken
bg-neutral-100 (hover)→hover:bg-surface-hover
border-neutral-200→border-line
border-neutral-300→border-line-2
text-neutral-900→text-ink
text-neutral-700/800→text-ink-2
text-neutral-500/600→text-ink-3
text-neutral-400→text-ink-4
text-red-600 / bg-red-600→text-danger / bg-danger (or <Button variant="danger">)
bg-blue-600 text-white (primary)→<Button variant="primary">
focus:ring-blue-400/500→focus:border-ink-3 (no ring)
rounded-md / rounded-lg→rounded (3px) / rounded-lg (6px)
shadow-sm / shadow-md / shadow-lg→shadow-card / shadow-card / shadow-pop (no mid tier)
font-semibold text-lg (heading)→font-serif font-medium text-[18px]
text-sm (body prose)→font-serif text-[13.5px] leading-[1.55]
text-xs (metadata)→font-mono text-[11px] text-ink-4
